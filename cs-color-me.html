The Problem

Designers spend too much time searching for harmonious color palettes that work well together. The process of finding color inspiration, testing combinations, and ensuring accessibility compliance was fragmented across multiple tools and websites, slowing down the design process.

The Solution

Created Color.me — a streamlined web tool that generates color harmonies using color theory principles, allows quick palette testing, and provides accessibility information, all in one interface.

Impact

  • Personal productivity increase of 60% when creating style guides
  • 200+ designers using the tool weekly after soft launch
  • Average session time of 8 minutes with high return usage
  • Integrated into workflow of 3 design agencies after beta testing

Initial Observations

What I experienced in my design workflow:

  • Constantly switching between multiple color tools and inspiration sites
  • Difficulty finding palettes that actually work well together in UI contexts
  • Time wasted on trial-and-error color combinations
  • No single tool that combined color theory with practical design application

Core problems identified — Fragmentation & Efficiency

Fragmentation Problem

  • Personal pain point: "I'm using 4 different tools just to pick colors for one project"
  • Fellow designers: "Adobe Color is powerful but overwhelming, Coolors is fun but not theory-based"
  • Design teams: "Everyone picks colors differently — we need consistency in our process"

Efficiency Problem

  • Time sink: Spending 30+ minutes on color selection that should take 5 minutes
  • Decision fatigue: Too many options without clear guidance on what works
  • Accessibility afterthought: Checking contrast ratios as a separate, final step

Early assumptions

  • Simplicity over features: Designers need focused tools, not feature-heavy applications
  • Theory-based approach: Color harmony rules can guide better design decisions
  • Integrated workflow: Combining generation, testing, and validation saves time

Market Research

Design tool landscape

  • $2.3B design software market with growing demand for specialized tools
  • 87% of designers use multiple tools for color-related tasks
  • Rise of design systems creating need for systematic color approaches

Existing solutions analysis

  • Adobe Color: Comprehensive but complex, part of expensive suite
  • Coolors.co: Popular but lacks color theory foundation
  • Paletton: Theory-based but outdated interface
  • Material Design: Good principles but limited to Material aesthetic

Market gap identified

No tool effectively combines color theory education with practical palette generation while maintaining a clean, focused interface that integrates into modern design workflows.

Becoming My Users

Research methods

  • Self-observation: Documented my own color selection process over 2 months
  • 8 designer interviews about color workflow pain points
  • Design community survey (45 responses) on social media
  • Workflow shadowing with 3 designers during actual projects

  • Experience: 3–5 years in digital product design
  • Goal: Create cohesive color systems for design projects quickly
  • Pain: "I bookmark color palettes but they never work in my actual designs"
  • Behavior: Uses multiple inspiration sites, tests colors in Figma/Sketch
  • Quote: "I need colors that actually work together, not just look pretty"

Secondary user: Alex, Design System Lead

  • Experience: 5+ years, manages design systems for teams
  • Goal: Establish consistent color selection methodology
  • Pain: "Team members all use different approaches to color"

Key insights

  • Color theory knowledge varies widely among designers
  • Inspiration vs. application gap — colors that look good in isolation don't always work in UI
  • Speed is critical — designers don't want to learn complex tools
  • Accessibility is often forgotten until final review stages
  • Consistency matters more than perfect colors

User journey pain points

  • Finding inspiration: Browsing multiple sites for color ideas
  • Testing combinations: Manual process of trying colors together
  • Checking accessibility: Separate tool for contrast checking
  • Documenting choices: Copy/pasting hex codes across tools

Competitor Research

Direct competitors analyzed

  • Adobe Color: Feature-rich but overwhelming, requires Creative Cloud subscription
  • Coolors.co: Simple and popular but lacks color theory foundation
  • Paletton: Strong color theory but poor user experience
  • Khroma: AI-powered but limited customization options

Key findings

  • No tool balances theory with simplicity effectively
  • Most focus on inspiration rather than practical application
  • Accessibility features are add-ons, not integrated
  • Export/integration capabilities vary widely

Our opportunity: Create a theory-informed, application-focused color tool that prioritizes speed and practical design use cases over inspiration browsing.

Strategic positioning: "The color tool that teaches you while you work — making better color decisions faster."

Solution

Design principles

  • Theory-informed decisions — Use color harmony rules to guide choices
  • Speed over options — Fast generation with smart defaults
  • Context-aware — Show colors in realistic design contexts
  • Education through use — Learn color theory while working

Key design decisions

Interface architecture

  • Single-screen workflow: All tools (generator, picker, palette) visible simultaneously
  • Color theory integration: Harmony type selector drives palette generation
  • Real-time preview: Generated palettes shown as color swatches with hex codes and contrast ratios

Core functionality

  • Base color selection: Color picker or hex input as starting point
  • Harmony type dropdown: Monochromatic, Triadic, Complementary options
  • Mood selector: Light/dark variants affect saturation and brightness
  • Random generation: Quick exploration with "Random" button

User experience optimization

  • Immediate feedback: Color changes reflect instantly in palette
  • Accessibility integration: Contrast ratios shown for each color automatically
  • Export simplicity: Click to copy hex codes, no complex export flows

Testing approach

  • Rapid prototyping with designer friends using actual projects
  • A/B tested harmony algorithm parameters for better aesthetic results
  • Usability sessions focused on speed of common tasks

Final Designs

Core interface features

Main Interface

  • Base color input: Large, prominent color picker and hex code field
  • Harmony type selector: Dropdown with color theory options (Monochromatic, Triadic, etc.)
  • Mood control: Light/Dark toggle affecting palette brightness
  • Random generation: Blue "Random" button for quick exploration

Generated Palette Display

  • 4-color harmony: Primary colors displayed as large swatches
  • Hex codes: Clearly labeled under each color for easy copying
  • Contrast ratios: Accessibility information integrated below each swatch
  • Visual hierarchy: Colors arranged left-to-right by dominance

Additional Features

  • Neutral suggestions: Warm and cool neutral options below main palette
  • Copy interaction: Click hex codes to copy to clipboard
  • Responsive design: Works effectively on desktop and tablet devices

Design system

  • Clean, minimal interface: Focus on color content, not interface chrome
  • Consistent spacing: 16px grid system for predictable layout
  • Typography: Simple, readable font hierarchy
  • Interaction feedback: Subtle hover states and click confirmations

Technical implementation

  • Color theory algorithms: HSL-based calculations for harmony generation
  • Real-time updates: Instant palette regeneration on input changes
  • Performance optimized: Fast rendering for smooth user experience

Reflection

Results (6 months post‑launch)

  • Personal workflow improvement: 60% faster style guide creation
  • User adoption: 200+ weekly active users from design community
  • Session engagement: 8-minute average sessions with 65% return rate
  • Professional adoption: 3 design agencies integrated into workflows
  • Community feedback: 4.3/5 rating from user surveys

What worked well

  • Color theory foundation gave users confidence in their choices
  • Single-screen interface eliminated tool-switching friction
  • Immediate visual feedback made experimentation enjoyable
  • Accessibility integration helped users build inclusive designs from the start

Key challenges

  • Balancing simplicity vs. power — resisted adding features that would complicate core workflow
  • Color algorithm tuning — required multiple iterations to generate aesthetically pleasing results
  • Cross-browser consistency — color rendering differences required careful testing

What I'd do differently

  • Earlier user testing with actual design projects rather than isolated tasks
  • Mobile optimization — didn't initially consider mobile design workflows
  • Export features — could have added more integration options with design tools
